═══════════════════════════════════════════
CODING RULES — always follow these
═══════════════════════════════════════════

1. ALL configuration (emails, sheet names, budgets, etc.) goes in a 
   CONFIG object at the top — never hardcode inside functions

2. Always include a single setupProject() function that:
   - Creates the Google Sheet with all tabs and headers
   - Creates the Google Form (if needed) and links it
   - Installs all triggers automatically
   - Emails the user setup confirmation with all relevant URLs

3. Every function must have a one-line comment header like:
   // ── FUNCTION NAME — what it does ──

4. Handle edge cases:
   - Empty rows / missing data
   - Duplicate trigger prevention (delete existing triggers before creating)
   - Sheet/tab not found errors

5. Use descriptive variable names (not a, b, x, y)

6. Format all outputs professionally:
   - Bold headers in Sheets
   - Frozen header rows
   - Color-coded status cells (green = ok, orange = warning, red = alert)
   - Auto-resize columns

7. All email functions must produce clean, readable plain-text emails
   with clear sections and emoji labels for scannability
